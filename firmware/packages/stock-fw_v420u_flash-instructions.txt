This is the official MeatPi stock firmware for the WiCAN-USB-C3.
Use this to set up the device for the first time or to recover from
a failed flash before installing openrs-fw.

  WHAT YOU NEED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Hardware:
  - MeatPi WiCAN-USB-C3 (on your desk, NOT in the car)
  - Mini USB data cable (must carry data — not charge-only)
  - Windows PC

  Software:
  - ESP32C3 Download Tool V3.9.0 (free, no install needed)
    → https://www.espressif.com/en/support/download/other-tools
    → Look for "Flash Download Tools" and download the zip
    → Extract and run the .exe

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  STEP 1 — ENTER BOOTLOADER MODE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  The WiCAN-USB-C3 does NOT have an exposed BOOT button.
  You must SHORT the two BOOT pads on the PCB using metal probes
  (tweezers, a small flat-head screwdriver tip, or jumper wires).

  PCB layout (top view):

    ┌──────────────────────────────────────┐
    │  [CANH] [CANL] [TR] [GND]           │
    │           ┌─────────────┐           │
    │           │  ● ●  BOOT  │ ← short   │
    │           │   pads      │   these   │
    │           └─────────────┘           │
    │                             [Mini USB]
    └──────────────────────────────────────┘

  1. Do NOT plug in the USB cable yet
  2. Locate the two small BOOT pads on the PCB (near the top,
     beside the ESP32-C3 module)
  3. Hold metal probes / tweezers across BOTH pads to short them
  4. While keeping the short, plug the Mini USB cable into the
     WiCAN and into your computer
  5. Keep the short for 1-2 seconds, then release
  6. The orange LED lights up — bootloader mode is active ✓

  What to use: metal tweezers work best. A flat-head screwdriver
  tip, two jumper wire ends, or any conductive metal across both
  pads simultaneously will work.

  If the orange LED does not appear: unplug, wait 3 seconds, and
  repeat. The pads must be shorted at the exact moment USB power
  is applied.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  STEP 2 — CONFIGURE THE FLASH TOOL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Startup dialog:
    ChipType:  ESP32-C3
    WorkMode:  develop
    LoadMode:  USB

  SPIDownload tab — load all 4 files:
    ☑  bootloader.bin                      @ 0x0
    ☑  partition-table.bin                 @ 0x8000
    ☑  ota_data_initial.bin                @ 0xd000
    ☑  wican-fw_usb_v420u_beta-01.bin      @ 0x10000

  SPI Flash config (IMPORTANT — use exactly these values):
    SPI SPEED:  40MHz    (not 80MHz)
    SPI MODE:   DIO      (not QIO)
    ☑ DoNotChgBin

  COM port and baud:
    COM:   select your device port (Device Manager → Ports)
    BAUD:  115200

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  STEP 3 — FLASH
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. Click START
  2. Green progress bar fills — takes about 30–60 seconds at 115200
  3. FINISH message appears when complete

  If recovering a bricked device: click ERASE first, wait, then START.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  STEP 4 — VERIFY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. Unplug the Mini USB cable
  2. Plug the WiCAN into the Focus RS OBD-II port (ignition ON)
  3. Wait 10–20 seconds for the LED to stabilise
  4. On your phone: Settings → Wi-Fi → connect to WiCAN_XXXXXX
     Password: @meatpi#
  5. Open http://192.168.80.1 — the WiCAN web UI loads ✓
     Firmware version shown should be v4.20u

  AFTER FLASHING — REQUIRED SETTINGS FOR openRS_ APP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  In the web UI at http://192.168.80.1 → Settings:
    WiFi Mode:  AP (Access Point)
    CAN Speed:  500 kbps
    Sleep Mode: Enabled  (voltage threshold ~13.1V)

  openRS_ connects via WebSocket (ws://192.168.80.1:80/ws) — the
  Protocol/Port settings in the web UI do not affect the app.
  No additional configuration needed.
